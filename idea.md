# Magnus 100 Food Tracker

Status: original project seed. The current implementation direction has evolved; use `docs/state.md`, `docs/decisions.md`, `docs/build-plan.md`, `docs/api-contract.md`, `docs/normalization.md`, `docs/qa.md`, and `docs/airtable-setup.md` as the authoritative planning docs.

Goal: build a small personal tool to help track meals, food habits, or nutrition-related notes for the Magnus 100 project.

Possible ideas:
- Log meals quickly from plain text
- Track recurring foods and simple patterns
- Summarize weekly food notes
- Turn messy meal notes into a clean table or checklist
- Export entries to CSV for later analysis

Constraints:
- Keep it small and local-first
- Prefer simple files like Markdown or CSV before adding a database
- Make the first version useful in one sitting
- I am learning Codex, so explain changes clearly before editing
