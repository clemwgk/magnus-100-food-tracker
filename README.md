# Magnus 100 Food Tracker

An installable, mobile-first PWA for tracking the first date Magnus tried each distinct ingredient. It uses a GitHub Pages frontend and a Google Apps Script proxy; Airtable credentials remain only in Script Properties.

## Local development

Use `npm.cmd` in PowerShell on this machine because the PowerShell execution policy blocks `npm.ps1`:

```powershell
npm.cmd ci
npm.cmd run dev
```

The app opens with Settings visible until an Apps Script **`/exec`** URL is saved locally. The passcode lives in memory only for the current tab session. The locally cached snapshot is used only for a quicker preview; it is not cached by the service worker.

## Phase 0: deploy the Apps Script proxy

1. In [Google Apps Script](https://script.google.com), create a standalone project named `Magnus 100 Food Tracker Proxy`.
2. Create `Code.gs`, `airtable.gs`, `auth.gs`, and `normalization.gs`; copy the matching files from [`apps-script`](apps-script). Replace the generated manifest with [`apps-script/appsscript.json`](apps-script/appsscript.json).
3. In **Project Settings → Script Properties**, add these exact keys. Enter values directly in the Apps Script UI—do not put them in a local file, the browser, or chat:

   | Property | Value |
   | --- | --- |
   | `AIRTABLE_TOKEN` | The restricted Airtable personal access token already stored in your password manager. |
   | `AIRTABLE_BASE_ID` | The `app...` ID for the configured base. Use a disposable base while running live checks. |
   | `FAMILY_PASSCODE_SALT` | Generated by `npm.cmd run make:passcode-hash`. |
   | `FAMILY_PASSCODE_HASH` | Generated by `npm.cmd run make:passcode-hash`. |

4. Run `npm.cmd run make:passcode-hash` in an interactive local terminal, choose the shared passcode, then copy only the two generated values into Script Properties. The helper hides typed input and never writes the passcode.
5. Deploy → **New deployment** → Web app. Set **Execute as: Me** and **Who has access: Anyone**. Copy the deployment URL ending in `/exec`. Never use `/dev` in the PWA.
6. Paste the `/exec` URL into the PWA’s Settings. It is public-sensitive but not a credential.
7. Every code change needs **Deploy → Manage deployments → Edit → New version** before retesting `/exec`.

The source has one protected test-only action, `verifyTestTarget`; it compares the configured base ID with a caller-supplied disposable-base ID without returning the configured ID. This prevents the live harness from mutating an accidentally configured family base.

## Required tests

```powershell
npm.cmd run lint
npm.cmd run test
npm.cmd run test:e2e
npm.cmd run secret:scan
npm.cmd run build
```

Live pre-build checks are intentionally closed by default. Create a **disposable Airtable base** matching the exact five-field schema, deploy a separate Apps Script project/configuration pointing at it, and set values only in the current PowerShell process:

```powershell
$env:RUN_LIVE_PREBUILD_TESTS='true'
$env:APPS_SCRIPT_URL='https://script.google.com/macros/s/.../exec'
$env:TEST_FAMILY_PASSCODE='your passcode'
$env:TEST_AIRTABLE_BASE_ID='app...'
$env:TEST_AIRTABLE_TOKEN='the disposable base token'
npm.cmd run test:prebuild
```

`TEST_AIRTABLE_TOKEN` is required solely to inspect the disposable schema and remove test rows. It must be restricted to that disposable base. The test refuses to mutate the proxy unless `verifyTestTarget` confirms that its Script Property base ID is precisely `TEST_AIRTABLE_BASE_ID`.

## GitHub Pages deployment

Set the repository’s Pages source to GitHub Actions, then use a workflow that runs `npm ci`, `npm run lint`, `npm run test`, `npm run secret:scan`, and `npm run build`; upload `dist`. Add the project path before building:

```powershell
$env:VITE_BASE_PATH='/REPOSITORY-NAME/'
npm.cmd run build
```

Do not set any Airtable or passcode values as frontend build variables. The PWA can also be deployed from `dist` through the Pages UI if Actions is unavailable.

## Manual acceptance

Follow the eight-step manual script in [`docs/llm-build-handoff.md`](docs/llm-build-handoff.md), including two iPhones, backdating carrot, an offline retry, and a private-browser source/network inspection. Use the real family base only after the disposable-base checks pass.
